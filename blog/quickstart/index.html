插件选择
#

必装扩展：统一代码风格、格式化、语法检查
#

  1. ESLint：javascript/typescript/json 语法检查

  2. Stylelint: css/scss/less 语法检查

  3. Prettier：格式化

  4. Vetur：智能提示、格式化、语法检查

选装扩展：更好的开发体验
#

  1. Auto Close Tag：自动追加末尾闭合标签

  2. Auto Renme Tag：重命名标签首尾同时修改

  3. CSS Navigation：跳转 HTML 的 class 属性定义

  4. File Peek：跳转文件引用

  5. Git History：查看 Git 提交历史记录

  6. GitLens：鼠标光标聚焦的代码行，行右侧将显示上一次提交信息

  7. Highlight Matching Tag：高亮左侧空白竖线和 HTML 标签下划线，便于观察首尾位置

  8. HTML CSS Support：提示 CSS 代码块

  9. Markdown All in One：扩展 markdown 特性，如创建目录、代码片段、快捷键等

  10. Markdown PDF：将 markdown 文件导出为 pdf 文件

  11. Markdown Preview Mermaid Support：使 markdown 支持流程图、思维导图等

  12. Path Autocomplete：自动补全引用的路径

  13. SVG：预览 SVG 图片

  14. Vue 3 Snippets：提示 Vue 代码块

  15. Vue Peek：跳转组件定义

  16. vue-helper：提示 Vue 代码块

注意：查看 .vscode/settings.json，若文本置灰，则表示未安装扩展，配置无效

开发调试
#

源码准备
#

  • 拉取代码

    git clone ${git地址}

  • 进入项目中

    cd ${项目目录}

    Windows 禁止忽略大小写

    由于 Windows 默认忽略文件夹及文件大小写，而 Linux/MacOS 不忽略，所以使 Windows 与其保持一致

    git config core.ignorecase false --global

# 代码规范

命名规则
#

  • Pascal 命名（AbCd）：class/type/interface/enum 名、自定义组件文件名和标签名、路由 name，样式(仅 ID)

  • Kebab 命名（a-b-c-d）：原生标签、第三方标签、标签属性，样式(ID 除外)

  • Camel 命名（aBcD）：其余均用

  • 路由 name 使用名词词组命名

  • 函数名尽量不要加 handle，思考使用更贴近当前业务应该用的动词开头，比如 openXXX，changeXXX，switchXXX，pickXXX 等

  • 命名单词尽可能思考简短贴近业务意思的单词，除了国际标准的缩写，尽量不要缩写，建议最多 5 个，超过 5 个则会降低可读性

约定建议
#

HTML 相关
#

  • 自定义通用组件使用 Aq 前缀，区别于历史的组件管理

  • 除了封装组件，禁止使用 style 属性写行内样式

  • 属性的编写顺序，按照重要程度的优先级排列：v- 前缀 > id/ref > 数据变量 > 效果变量 > 其他属性 > 事件(最末尾)，如下：

<input v-if="visible" v-model="value" :max="maxLength" :readonly="isReadonly" clearable @input="searchKeyword"></input>

JS/TS 相关
#

  • 优先思考可否使用值类型定义变量和初始化变量；实在增加工作量，才考虑用 any 定义，null 初始化

  • 优先考虑可否仅用 if 做判断，实在需要 else 才加，尽量避免使用 else if，利于减少嵌套层级，可读性更好

  • 使用 switch 时，每个 case 代码块尽量不要超过 10 行，超过 10 行应该思考写成方法，这样可读性更好

  • 使用 for 而不是 forEach，防止内部回调存在异步问题，注意 for / for in / for of / forEach / map 的区别，按场景使用

  • 使用 filter 过滤首条，而不是 find，防止 undefined 导致异常

  • 使用路由的 name 跳转，而不是 path，减少和平台的绑定关系

  • loading 加载一定要在 finally 中关闭，防止异常导致无法关闭

  • 接口传参尽可能定义为值类型，可读性更好，规避接口提供冗余字段难以查找，新增表单和更新表单可以除外

  • 仅键值对初始化为{}，其他 object 类型，初始化均用 null，禁止用 undefined

    PS：由于历史原因未在 eslint 规则中强制限制

  • 能用 this，尽量用 this，若作用域有问题，导致报错，才使用变量赋值 this

CSS/SCSS/LESS 相关
#

  • 优先思考可否使用自适应布局样式

  • 覆盖第三方样式：尽量不使用 v-deep、/deep/ 和 ‘»>’，会偶现不生效

    • 可使用两个 style 标签，含 scoped：仅自定义样式，不含 scoped：仅覆盖第三方样式

    • 优先思考优先级覆盖，实在不行，可使用!important
